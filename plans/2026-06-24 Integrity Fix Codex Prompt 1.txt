The worker must not depend on that accidental difference.

Goal:

Fix the v2 AQI-only rebuild worker so connector-scoped rebuilds are valid long-term.

Required semantics:

1. Full-day v2 AQI rebuild, no connector filter

When no connector filter is supplied, the worker should rebuild all observed connectors for the selected day/window, as it does today.

2. Connector-scoped v2 AQI rebuild, connector filter supplied

When a connector filter is supplied, for example:

```text id="czzgda"
UK_AQ_BACKFILL_CONNECTOR_IDS=1
```

the worker should:

* Check whether connector 1 has v2 observations for the day.
* If connector 1 exists, rebuild connector 1.
* Ignore other observed connectors such as 3 and 7 for the purpose of deciding whether connector 1 is allowed to run.
* Not fail with `missing_connectors=[3,7]`.
* Not accidentally rebuild connectors 3 and 7.
* Not delete or overwrite other connectors’ existing AQI data.
* Preserve or merge existing day/connector/pollutant manifests/indexes correctly if the code maintains aggregate manifests.

3. Requested connector missing

If a connector filter is supplied and the requested connector has no v2 observations for the day, then the worker should skip/fail that requested connector with a clear diagnostic, for example:

```text id="dsg2w1"
target_connector_observations_missing
```

or similar.

It should not report unrequested connectors as missing.

4. Manifest/index behaviour

Investigate how v2 AQI data manifests are structured. Current R2 layout includes connector-scoped pollutant directories:

```text id="tkg4cn"
history/v2/aqilevels/hourly/data/day_utc=<day>/connector_id=<connector>/pollutant_code=<pollutant>/
```

The fix should respect that connector/day/pollutant scope.

If there is an aggregate day manifest or connector manifest that is rebuilt, then for connector-scoped repair the code must either:

* rebuild only the affected connector manifest/index, or
* merge/preserve existing entries for other connectors, or
* regenerate the aggregate manifest from the actual R2/local tree without requiring unrequested connectors to be rebuilt in the same run.

Do not make connector-scoped repair dependent on existing AQI rows for other connectors.

What not to do:

Do not remove the guard entirely. Replace it with the correct guard:

```text id="jfi7o3"
For connector-filtered rebuilds, verify requested connectors are present in observations.
Do not require every observed connector on the day to be included in the connector filter.
```

Do not make the integrity runner queue full-day AQI rebuilds just to avoid this. Connector-scoped repair is required.
